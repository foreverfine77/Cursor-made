## 二、经验
1. 项目开始好像怎么问都行。
2. 中间修改的时候，最好一小点一小点的问，这样可以更好更改。
3. 更新代码最好使用save all，先查看有无明显出错的地方。
4. 因为时常修改后会改其他部分，这时需要拖动滚动条，看红条绿条标出的部分，就可以快速发现修改出错的地方。
